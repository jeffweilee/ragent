| 2026-05-04 | **Architecture** | Worker deleted MinIO staging object **before** committing terminal `status=READY`, so a crash in the gap caused the next reconciler retry to 404 on MinIO and eventually mark the document `FAILED` despite a successful pipeline run. Mirror risk: supersede ordered by `created_at < self` was not order-safe — a newer doc finishing READY before an older doc left the older one orphaned. | Side-effect cleanup (and stateful comparisons) ordered before / against durable state commits. | **[Rule]** Within any worker, **durable state commits MUST precede external side-effect cleanup**: (1) commit terminal status; (2) best-effort cleanup of staging artefacts; (3) log orphan-tolerated event. For convergence rules across multiple writers (e.g. supersede), use a **set-based survivor selection** (`MAX`/`MIN` over the candidate set under `FOR UPDATE SKIP LOCKED`) rather than a self-referential pairwise comparison. Applies to MinIO, Redis staging, and any future transient store. |
| 2026-05-04 | **Resilience** | Long-running pipeline held `SELECT … FOR UPDATE` on the document row across the entire 60+ s pipeline body (embedder, ES, plugins). A live-but-hung worker would never release the lock, and the Reconciler's `SKIP LOCKED` would skip the row forever — silent stuck-PENDING. Worker process *death* releases the lock (DB drops session); worker *hang* does not. | Single transaction wrapped both the cheap status mutation and the expensive external pipeline body. | **[Rule]** Status mutations and external work MUST use **separate short transactions**. Acquire `FOR UPDATE NOWAIT`, write the new status, **commit**, run external work outside any DB tx, then re-acquire briefly to commit the terminal status. Never hold a row lock across an external call. Worker uses `NOWAIT` (re-kiq self on contention); only the Reconciler uses `SKIP LOCKED`. |
| 2026-05-04 | **Resilience** | Recovery surface was incomplete: Reconciler swept only `PENDING/DELETING > 5 min`. UPLOADED orphans (TaskIQ message lost), multi-READY rows (supersede task lost between `status=READY` commit and kiq), and a missing reconciler heartbeat were all silent invariant violations with no automated repair. | New transient states / convergence rules added without extending the Reconciler in lockstep. | **[Rule]** Every status that is **not terminal** and every **eventual-consistency invariant** must have a Reconciler arm. When introducing a new transient state (e.g. UPLOADED, DELETING) or an async convergence task (e.g. supersede), add: (a) a sweep query keyed on age or invariant violation, (b) a BDD scenario, (c) a plan row. Reconciler itself emits a `reconciler_tick_total` heartbeat with a Prometheus alert at > 10 min absence (no watcher of the watcher). |
| 2026-05-04 | **Performance** | Spec said `{ESVector ∥ ESBM25}` and used parallel-bar notation, but Haystack sync Pipeline runs components sequentially — actual chat latency was additive, not the implied max. Embedder catalog described retries but no batching, so a 500-chunk doc would issue 500 HTTP calls. | Notation borrowed from a parallel-runtime spec was applied to a sequential runtime; batching defaulted to 1 because the contract did not specify. | **[Rule]** Concurrency notation (`∥`, `par`, "concurrent", "fan-out") in the spec must be matched by a **runtime that actually runs in parallel** in that phase, or annotated `(sequential in PX)`. Every call to a remote service that accepts arrays must declare an explicit batch size in the catalog (`batch=N`). Per-call timeouts are mandatory in §4.3 / §4.5; "implicit default" is not acceptable. |
| 2026-05-04 | **Spec discipline** | Implementation about to start with 10 implicit decisions still floating in conversation (chunker choice, CSV scaling, migration tool, health endpoints, error format, SSE error wire-shape, pagination semantics, test backends, scheduler runtime, MinIO key). Each would have produced incompatible code across tracks if discovered ad-hoc during TDD. | Spec captured *what to build* and *invariants*, but treated *contracts at integration boundaries* (HTTP error shape, SSE framing, object-store keys, scheduler runtime) as "obvious" — they were not. | **[Rule]** Before any track starts Red phase, run a **Decision-Surface audit** against the spec: every external-facing contract (HTTP body shape, streaming framing, object keys, scheduler runtime, test fixture provider, schema-migration mechanism) must have an explicit answer in spec, or a row in `§7. Decision Log` with rationale and rejected alternatives. Implicit defaults are blocking — no Red phase begins until the row is filled. |
| 2026-05-04 | **Architecture** | Earlier round 3 declared "OpenFGA out-of-scope across all phases" and proposed an owner-based ES filter (`terms(owner_user_id ∈ [user_id])`) as the P2 ACL. That decision is **reversed** in B14: the owner-on-ES approach couples auth to the retrieval index (re-index on every permission-model change), forces a single-relation model (no sharing/groups/roles), and overloads the column name `owner_user_id` with authorization semantics it was never going to keep. Mid-round reversal cost re-edits across §1, §3.4, §3.5, §4.1, §5.1, the §7 Decision Log row B7, and the entire T8 plan track. | Auth-shape decisions were taken under "minimum-viable for P2" pressure without weighing the lifetime cost of putting auth fields into the retrieval index. The "no out-of-band ACL service" framing was treated as a hard constraint when the real constraint was "no per-query out-of-band call inside the retrieval hot path" — which a post-retrieval gate satisfies. | **[Rule]** Auth/permission fields **MUST NOT** live on the retrieval index (ES, vector store, graph store). Permission gating is a **post-retrieval layer** behind a Protocol; the backend (OpenFGA, opaque DB lookup, etc.) is swappable without re-indexing. Column names in the data layer must describe **what they record** (`create_user` = creator audit), never **what they authorize** (`owner_user_id` implied an ACL role the column did not enforce). When reversing a prior Decision-Log row, append a new dated row that explicitly cites the superseded ID — never edit the old row in place. |
